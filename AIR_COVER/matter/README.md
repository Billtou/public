## 1. 產品簡介

AirCover 是 AUTOMATE 推出的智能捲簾控制升級套件 — **替換 Tuya 捲簾 / 窗簾控制器原廠的 CB3S Wi-Fi 模組**,讓本來只能用 Tuya/Smart Life App 控制的傳統 Tuya 捲簾,**直接加入 Matter 智能家庭生態**,不需專屬 Hub 也不必雲端帳號。

主要特性:
- **無痕升級** — 接腳完全相容 CB3S,焊上即用,不用換馬達 / 軌道 / 控制器
- **Matter 標準協定** — 直接加入 Apple Home / Google Home / SmartThings / Home Assistant / Amazon Alexa
- **跨生態同步** — 同一台裝置可同時加入多個生態(最多 5 個 Fabric)
- **精確位置控制** — 0–100% 任意位置(視原廠馬達能力)
- **本地控制** — Wi-Fi / Hub 斷線時仍可用實體按鈕操作,不靠雲端
- **與原廠遙控 / App 並存** — 你既有的實體遙控、Tuya App、本地按鍵都還能用

### 1.1 規格速覽

| 項目 | 內容 |
|---|---|
| 通訊協定 | Matter 1.x over Wi-Fi |
| 控制協定 | Tuya MCU UART(原廠通訊規格 9600 8N1) |
| Wi-Fi | 2.4 GHz 802.11 b/g/n(**不支援 5 GHz**) |
| BLE | 5.0(僅配對用) |
| 處理器 | ESP32-C3(RISC-V 160 MHz) |
| Flash | 4 MB |
| 模組形式 | 取代 Tuya CB3S 模組,接腳相容 |
| 供電 | 由原機台 3.3V VCC 供電(<100 mA) |
| 工作環境 | -10 – 60 °C / 10 – 90% RH(無凝結) |

### 1.2 包裝內容

- AirCover 模組(已焊好排針)× 1
- 快速入門卡(含 Matter QR Code 與 11 碼配對碼)× 1

### 1.3 相容性

✅ **可用**:Tuya 通訊協定的捲簾 / 窗簾 / 百葉控制器(DP1=控制 / DP2=百分比 / DP3=位置回報 / DP5=方向)
- 多數 Tuya 認證的捲簾馬達 / 軌道 / 百葉
- 常見 LoraTap、Zemismart、Aubess、Moes 等品牌的 Tuya 版本

❌ **不支援**:
- Zigbee 版本(完全不同協定)
- Wi-Fi 但不是 Tuya 系統(例如 SmartLife OEM 但走 Aqara、Yeelight 自家協定)
- 紅外控制 / 433 MHz 控制器

**判斷方式**:看原機台拆開後的模組編號 — 若是 `CB3S` 字樣的方形模組就可用。

## 3. LED 狀態指示

### 3.1 4 種運作狀態

依優先級從高到低:

| 優先級 | 狀態 | LED 行為 | 含義 |
|---|---|---|---|
| 1 | 重置警告 | 5 Hz 快閃(亮 0.1s 暗 0.1s) | 按鍵已按 5 秒,再 5 秒就執行工廠重置 |
| 2 | 等待 Wi-Fi | 閃 1 下 + 停 1.8s | 尚未設定 Wi-Fi / Wi-Fi 連不上 |
| 3 | 未配對 / 配對中 | 雙閃 + 停 1.4s | Wi-Fi 已連,Matter commissioning 進行中或還沒加入任何生態 |
| 4 | 正常運作 | 熄滅 | 一切就緒 |

### 3.2 快速判斷對照表

| 看到 LED | 通常代表 | 該做什麼 |
|---|---|---|
| 閃 1 下 | 沒設定過 Wi-Fi / 重置後 | 用手機 App 配對流程加入 |
| 雙閃 | Matter 配對流程進行中 | 等 30 秒;超過 1 分鐘還在雙閃 → App 重新加入 |
| 熄滅 | 已上線運作中 | 無需處理 |
| 快閃 | 你正在長按重置鍵 | 想取消重置就放開;想重置就按到 10 秒 |

## 4. 按鍵操作

> ⚠️ 本章節是 **AirCover PCB 板上的按鍵**,需要打開原機台外殼才能按。
> **日常 user 不需要動到** — 重置請優先用 [章節 10.1 原機台對碼鍵](#101-用原機台對碼鍵推薦免拆殼)(免拆殼)。

板上微動鈕(GPIO 1),大多時候靠遙控、App、語音、自動化。

### 4.1 短按:開關切換

| 動作 | 結果 |
|---|---|
| 按住 < 5 秒後放開 | 切換捲簾狀態 — 開→關 / 關→開 |

> 若目前在「半開」位置(0% < 位置 < 100%),按一下會依「目前距離 0% 較近就關、距離 100% 較近就開」邏輯。

### 4.2 長按:工廠重置

| 動作 | 結果 |
|---|---|
| 持續按住 0–5 秒 | 無反應(放開 = 短按切換) |
| 持續按住達 5 秒 | LED 開始**快閃**(警告中,放開可取消) |
| 持續按住達 10 秒 | **工廠重置觸發**,LED 熄滅,裝置自動重啟 |

> 重置範圍見 [章節 10.3](#103-重置會清除什麼)。

## 5. 接入智能生態(初次配對)

### 5.1 配對前確認

- [ ] 路由器 **Wi-Fi 2.4 GHz** 可用(AirCover 不支援 5 GHz)
- [ ] 手機 **藍牙開啟**,跟 AirCover 距離 < 1 公尺
- [ ] 手機**已加入** Matter 平台(Apple Home / Google Home / SmartThings / HA / Alexa)
- [ ] 至少有一台 Matter Controller 在線:
  - **Apple**:iPhone / iPad / HomePod / Apple TV(iOS 16.1+)
  - **Google**:Nest Hub / Android 14+ 手機
  - **Home Assistant**:主機 + iPhone / Android 14+ 手機
  - **Alexa**:Echo Hub / Echo (4th gen) / Echo Show

### 5.2 Apple Home

1. 通電 AirCover,等狀態 LED 開始**閃 1 下 + 停**
2. iPhone 開「家庭」App → 右上角 **+** → **加入配件**
3. 對準 QR Code 掃描;或選「沒有代碼/無法掃描」→ 輸入 `3642-630-6820`
4. iPhone 自動傳 Wi-Fi 憑證 → LED 變**雙閃**
5. 配對完成 → LED **熄滅**,出現「窗簾」裝置,可拉開關 / 設位置 / 加自動化

### 5.3 Home Assistant

1. 通電,等 LED 閃 1 下
2. 開 Home Assistant App → 設定 → 裝置與服務 → Matter → **加入 Matter 裝置**
3. 掃 QR 或輸入 `3642-630-6820`
4. App 自動完成 commissioning
5. 在「裝置」清單會看到 **automate-aircover-XXXXXX**,內含「窗簾」實體(cover entity)

### 5.4 Google Home

1. 通電,等 LED 閃 1 下
2. 開「Google Home」App → 右上 **+** → **設定裝置** → **新裝置** → 選擇家庭
3. App 自動偵測「Matter 裝置」→ 點擊 → 掃 QR 或輸入 `3642-630-6820`
4. 依 App 指示完成 Wi-Fi 設定
5. 配對完成,可在「窗簾」分類找到

### 5.5 Amazon Alexa

1. 通電,等 LED 閃 1 下
2. 開 Alexa App → 裝置 → **+** → **新增裝置** → **其他** → **Matter**
3. 掃 QR 或輸入 `3642-630-6820`
4. 完成

## 6. 跨生態接入(Matter Multi-Admin)

### 6.1 機制說明

Matter 規範允許**同一台裝置同時加入多個生態**,最多 **5 個 Fabric**。例如同時被 Apple Home 與 Home Assistant 控制,在任一生態下發「關窗簾」指令,實體都會動作。

**關鍵原則**:第二個生態起,**不能用出廠那組 `3642-630-6820`** — 必須由**第一個已配對的生態**產生一組「分享配對碼」(每次新的、有時效)。

### 6.2 從 Apple Home 分享給其他生態

1. 家庭 App → 長按窗簾裝置 → **設定**(齒輪圖示)
2. 滑到底,找 **「開啟配對模式」**
3. App 顯示一組 11 碼**拷貝代碼**(15 分鐘內有效)
4. 在第二個生態 App「加入配件」流程輸入這組新碼

### 6.3 從 Home Assistant 分享給其他生態

1. Home Assistant App → 設定 → Matter → 指定轉出裝置 → **分享裝置**
2. **複製代碼**
3. 在第二個生態 App「加入配件」流程輸入這組新碼

### 6.4 常見誤區

| 誤操作 | 後果 | 正確做法 |
|---|---|---|
| 第二個生態用出廠 `3642-630-6820` | 配對失敗 | 用第一個生態生出來的「分享碼」 |
| 用 Apple 重置裝置後再加入 Google | Apple 那邊也會掉 | 不要重置,用「Multi-Admin 分享」流程 |
| 已加入 5 個生態還想加 | 失敗 | 移除其中一個再加 |
| Wi-Fi 不通就嘗試多 admin | 失敗 | 先確認原配對生態正常,Wi-Fi 通才能分享 |

## 8. 捲簾控制

### 8.1 全開 / 全關 / 暫停

任何生態的窗簾介面都有 3 個基本動作:

| 動作 | iOS Home App | Home Assistant | 對應指令 |
|---|---|---|---|
| **全開** | 拉到 0% / 按「打開」 | `cover.open_cover` | Tuya DP1 = open |
| **全關** | 拉到 100% / 按「關閉」 | `cover.close_cover` | Tuya DP1 = close |
| **暫停** | 運轉中再按一次 | `cover.stop_cover` | Tuya DP1 = stop |

### 8.2 精確位置(0–100%)

iOS Home App 拉桿、HA `cover.set_cover_position` 都支援 0–100% 任意值:

- **0%** = 完全打開
- **100%** = 完全關閉
- **50%** = 半開

> 實際定位精度依原機台馬達能力 — 高階捲簾通常 ±1%,入門款可能跳格 5–10%。

### 8.3 語音控制

| 平台 | 範例 |
|---|---|
| **Siri** | 「Hey Siri,打開客廳窗簾」/ 「關到 30%」 |
| **Google** | 「OK Google,打開臥室捲簾」 |
| **Alexa** | 「Alexa, open the curtain」 |

### 8.4 自動化情境

捲簾在自動化中可當作 **動作 (Action)**:

- ⏰ **時間排程** — 早上 7:00 自動全開,晚上 19:00 自動全關
- 🌡 **氣溫聯動** — 夏天室內 > 28°C 自動關到 50% 遮陽
- 🌅 **日出日落** — 跟 iPhone 內建日出日落事件聯動
- 👤 **離家 / 回家** — 偵測到無人時關全部窗簾,回家前 10 分鐘自動打開

> 自動化在 iOS Home App / HA / Google Home 內各自設定,**不需 AirCover 額外配置**。

## 9. OTA 韌體更新

### 9.1 更新流程

1. 連線 Web UI → 底部 **版更** Tab
2. 系統自動向官方 GitHub 檢查最新版本
3. 若有新版,顯示「有新版本可更新」+ 「開始更新」按鈕
4. 點擊「開始更新」→ 確認對話框 → 自動下載
5. 下載完成自動重啟,Web UI 自動 reload 回首頁

整個過程約 1–2 分鐘,期間裝置會暫時離線,**捲簾位置不會亂動**。

### 9.2 更新會保留 / 清除什麼

| 項目 | 是否保留 |
|---|---|
| Matter 配對(所有生態) | ✅ 保留 |
| Wi-Fi 帳密 | ✅ 保留 |
| 捲簾目前位置 | ✅ 保留 |
| 韌體版本 | ❌ 換新版 |

## 10. 工廠重置

### 10.1 用原機台對碼鍵(推薦,免拆殼)

> 💡 這是最簡單的方法,**完全不用拆機殼**。

按你**原機台**(Tuya 捲簾控制器)上的「**對碼鍵 / 配網鍵 / WiFi 鍵**」— 通常在面板上有 RESET / WiFi 圖示的小孔(需用迴紋針 / 細針戳),原廠說明書內有指示。

**觸發方式依原機台 OEM 設定,常見三種**:

| 模式 | 操作 | 機器類型 |
|---|---|---|
| **連按 3 下 + 持續 2 秒** | 三下 → 第四下按住 2 秒 | 多數中階 Tuya 捲簾 |
| 長按 5–10 秒 | 持續按 5 秒以上 | 部分入門款 |
| 長按 + 看 LED 變化 | 等原機台 LED 變閃才放開 | 部分機器 |

**正確的觸發方式請查原機台說明書**(關鍵字:「重設 Wi-Fi」/「Reset Network」/「重新配對」)。

成功觸發後,AirCover 收到信號 → 自動清除所有 Matter 配對 → 3 秒重啟 → 進入可重新配對狀態。LED 從正常運作切換成**雙閃**(等待配對中)。

### 10.2 Web UI 重置

若 Wi-Fi 還在,可從瀏覽器操作:

- 開 `http://automate-aircover-XXXXXX.local/update`
- 「裝置資料清除」card → 「清除 NVS(出廠重置)」
- 確認對話框 → 自動重啟

### 10.3 硬體重置(需拆殼)

> ⚠️ 通常不需要 — 章節 10.1 已可達成同樣效果。**只有原機台對碼鍵壞掉**才用這條。

- 打開原機台外殼,找到 AirCover 模組上的多功能按鍵
- 按住 5 秒 → LED **快閃**(警告階段)
- 繼續按住到 **10 秒** → **工廠重置觸發**,LED 熄滅,裝置自動重啟

> 5–10 秒之間放手 = 取消(LED 回原狀態,不重置)。

### 10.4 重置會清除什麼

| 項目 | 是否清除 |
|---|---|
| Matter 配對(所有生態) | ✅ 清除 |
| Wi-Fi 帳密 | ✅ 清除 |
| 韌體版本 | ❌ 不變 |
| 裝置認證資料 / DAC | ❌ 不變(內建出廠憑證,不會掉) |
| 捲簾當下位置 | ❌ 不變(實體狀態,跟 AirCover 無關) |

## 11. 故障排除

### 11.1 配對相關

| 現象 | 原因 | 處理 |
|---|---|---|
| LED 一直閃 1 下,App 找不到裝置 | 配對流程沒走完 / 手機藍牙關閉 / 距離太遠 | 確認手機藍牙開、距離 < 1 m、Wi-Fi 是 2.4 GHz |
| 雙閃很久最後又回單閃 | 裝置中途重啟(供電不足) | 確認接線正確,VCC 穩定 |
| App 顯示「無法加入」 | 配對碼錯誤 / Fabric 已滿 | 重新輸入碼;若已用 5 個生態,先移除其一 |
| 第二個生態加不進 | 用了出廠那組碼 | 改用第一個生態的「分享碼」(章節 6) |
| **想換 Wi-Fi / 已從生態刪除想重來** | 需重置 Matter 配對 | 按原機台「對碼鍵」(章節 10.1)— 觸發方式依原機台說明書,通常是**連按 3 下 + 持續 2 秒** 或 長按 5–10 秒 |
| 原機台對碼鍵不知道在哪 / 按了沒反應 | 不同 OEM 的觸發 pattern 不同 / 廉價款省了對碼鍵 | 看原機台說明書「重設 Wi-Fi」段落;真的找不到才拆殼用 AirCover PCB 按鍵(章節 10.3) |

### 11.2 控制相關

| 現象 | 原因 | 處理 |
|---|---|---|
| 按開不動 / 按關不動 | 接線異常 / Tuya MCU 沒回應 | 拆殼檢查 UART 接線(TX/RX 沒接反) |
| 開到一半就停 | 原機台行程限位設錯 / 物理障礙 | 用原廠遙控學習行程後再控制 |
| 位置回報跟實際不同步 | 原機台 percent_state 沒回報 | 重置後重新校正,或長按「全開→全關」走完整行程一次 |
| HK 顯示「無回應」 | Wi-Fi 斷線 / 路由器太遠 | 確認 Wi-Fi RSSI > -70 dBm |
| 暫停按了沒效果 | 馬達已停 / 不支援即時暫停 | 取決於原機台馬達 — 部份廉價馬達不支援動作中暫停 |

### 11.3 OTA 相關

| 現象 | 原因 | 處理 |
|---|---|---|
| 版更頁顯示「無法取得版本」 | 裝置連不到 GitHub | 檢查網路,可能是 DNS / 防火牆問題 |
| 「開始更新」後一直 0% | 下載卡住 | 等 5 分鐘;不行就重啟裝置再試 |
| 更新失敗 | 韌體損毀 / 供電不足 | 重啟裝置(會回到舊版),確認供電後重試 |

## 12. 安全使用

- ⚡ **接線前務必斷電** — 原機台通常接 110/220V AC,模組焊接時必須整個拔插頭
- 🔌 **不可碰原廠 110/220V 部分** — AirCover 只接低壓 3.3V UART 端,馬達電源由原機台處理
- 🌡 工作環境溫度 -10 – 60 °C,**勿安裝於陽光直曬 / 廚房油煙環境**
- 🌧 戶外捲簾使用須確認原機台是防水等級,AirCover 模組不防水
- ⚠️ **改裝可能讓原機台失去保固** — 由消費者自行承擔
- 🔧 **建議由電器專業人員施工** — 焊接 / 拆卸請找有經驗的人

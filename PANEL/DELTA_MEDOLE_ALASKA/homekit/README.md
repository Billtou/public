<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">3. 初次設定(連 Wi-Fi + 配對)</h2>

### 3.1 讓面板連上家中 Wi-Fi

首次開機(或重置後)面板會自動開啟一個設定熱點:

1. 手機 → **設定 → Wi-Fi** → 連上 `automate-ctrl-panel-XXXXXX`(XXXXXX = MAC 後 6 碼)
2. 通常會自動跳出設定頁;沒跳出就用瀏覽器開 **`http://192.168.4.1`**
3. 選擇家中 **2.4 GHz** Wi-Fi、輸入密碼 → 儲存
4. 面板自動重啟並連上網路

<blockquote style="border-left:4px solid #ff6f48;background:#fff7f4;padding:14px 18px;margin:16px 0;border-radius:0 12px 12px 0;color:#5a3520">
⚠️ <b>務必選 2.4 GHz</b>。面板不支援 5 GHz —— 若路由器把 2.4/5 GHz 合併成同一個 SSID,設定期間請先暫時分開,或關閉 5 GHz。
</blockquote>

### 3.2 加入 Apple 家庭

1. 確認家中有 **Home Hub**(HomePod / Apple TV / 常駐家中的 iPad)—— HomeKit 必要
2. iPhone 開 **家庭 App** → 右上 **＋** → **加入配件**
3. 掃描**面板螢幕上**的 HomeKit QR Code(面板底部 QR 圖示進入配對頁)
4. 依畫面指示完成,約 30–60 秒
5. 命名與分配房間 → 完成

<blockquote style="border-left:4px solid #007AFF;background:#eff7ff;padding:14px 18px;margin:16px 0;border-radius:0 12px 12px 0;color:#1c3d5a">
💡 掃不到 QR 時可選「沒有代碼或無法掃描」→ 手動輸入 <code>111-88-999</code>。
</blockquote>

<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">4. Apple Home 控制</h2>

### 4.1 🔥 暖風機 / 🏔️ 阿拉斯加暖風機

家庭 App 顯示為**一個延長線磚塊**,點進去有四個插座:

| 插座 | 作用 |
|---|---|
| **暖房** | 加熱 |
| **涼風** | 送涼風 |
| **乾燥** | 衣物乾燥 / 除濕 |
| **換氣** | 排氣換氣 |

- **互斥** — 打開任一個,其餘自動關閉;四個全關 = 機器停止
- **室溫** — 暖風機另有獨立的室溫感測器(阿拉斯加無)
- **主燈** — 若訂購機型含主燈,會多一個「主燈」插座(可獨立開關,不受模式互斥影響)

### 4.2 🌬️ 全熱交換器

一個延長線磚塊,六個插座:**低速 / 中速 / 高速 / 自訂1 / 自訂2 / 自訂3**(互斥)。
另有**外氣**與**回風**兩個溫度感測器(視機型而定)。

> 自訂 1–3 的送風 / 排風百分比在**面板上**設定,Apple Home 只負責切換。

### 4.3 💨 管道風機

家庭 App 顯示為**風扇**:開關 + **0–100% 風量滑桿**。

<blockquote style="border-left:4px solid #007AFF;background:#eff7ff;padding:14px 18px;margin:16px 0;border-radius:0 12px 12px 0;color:#1c3d5a">
💡 Apple Home 的百分比與面板顯示的百分比<b>完全同步</b> —— 兩邊拖同一個值。
</blockquote>

### 4.4 💧 米多力除濕機

| 家庭 App 項目 | 作用 |
|---|---|
| **除濕機** | 開關 · 目標濕度 · 目前濕度 |
| **清淨** | 空氣清淨開關 |
| **換氣** | 送風 / 換氣開關 |
| **室溫** | 溫度感測器 |
| **漏水** | 水滿 / 漏水偵測(觸發時家庭 App 會通知)|

### 4.5 自動化建議

| 情境 | 設定 |
|---|---|
| 洗澡前暖房 | 「晚上 21:00」→ 打開暖房 |
| 洗完自動乾燥 | 「濕度高於 70%」→ 打開乾燥 |
| 定時換氣 | 「每 3 小時」→ 打開換氣 30 分鐘後關閉 |
| 漏水警報 | 米多力「漏水偵測」→ 推播通知 + 關閉除濕機 |

<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">5. 面板操作</h2>

面板螢幕本身就是完整的控制介面,**不需要手機也能用**。底部四個圖示切換頁面:

| 圖示 | 頁面 | 內容 |
|---|---|---|
| 機型圖示 | **主畫面** | 開關 / 模式 / 強弱 / 定時 / 溫濕度 |
| ⚙️ | **設定** | 螢幕亮度、螢幕睡眠、連網狀態、濾網時數、狀態、畫面刷新 |
| ▣ | **配對** | HomeKit QR Code、使用說明 QR、重置連網、重開機 |
| A/B | **機型切換** | 安裝人員用(需密碼)|

### 5.1 濾網提醒

設定頁可看到 **濾網 已用 / 門檻 小時**,按「濾網」可選提醒時數(1500 / 2160 / 3000 / 4000 小時),按「重置」把已用時數歸零。超過門檻時數字會轉紅。

### 5.2 畫面刷新

若螢幕出現位移或異常,按設定頁的「**刷新**」可重建畫面,不影響家電運作與配對。

<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">6. Home Assistant 整合 (MQTT)</h2>

面板內建 MQTT bridge,**Apple Home 與 Home Assistant 可同時使用**(雙生態,狀態雙向同步)。
適合已有 Mosquitto / HA 環境的進階使用者。

### 6.1 適用對象

| 你是 | 用不用得到 |
|---|---|
| 只用 Apple Home,沒 HA | 跳過本章,Apple Home 功能完整不缺 |
| 已有 Home Assistant + Mosquitto broker | 跟著本章設定,可在 HA 看到面板 entity |
| 沒裝 Mosquitto | 先在 HA 裝 `Mosquitto broker` add-on,設帳密再回來 |

<blockquote style="border-left:4px solid #ff6f48;background:#fff7f4;padding:14px 18px;margin:16px 0;border-radius:0 12px 12px 0;color:#5a3520">
⚠️ <b>目前支援 MQTT 的機型</b>:暖風機、全熱交換器、米多力除濕機。<br>
<b>管道風機與阿拉斯加暖風機尚未支援 MQTT</b>(Apple Home 控制不受影響),將於後續韌體版本加入。
</blockquote>

### 6.2 設定步驟

1. 用瀏覽器打開 **`http://<面板-IP>/engineer`**(面板 IP 從家庭 App → 點裝置 → 設定 → 看 IP,或從設定頁的連網狀態查)
2. 頁面捲到 **「MQTT / Home Assistant」** 卡片
3. 填入:
   | 欄位 | 範例 | 說明 |
   |---|---|---|
   | Broker IP | `192.168.1.10` | Mosquitto 主機 IP(一般填 HA 的 IP)|
   | Port | `1883` | 預設 1883 |
   | Username | `mqtt-user` | broker 帳號(在 HA 創立)|
   | Password | `********` | broker 密碼(在 HA 創立)|
4. 按 **「連線測試」** — 必須看到綠色「✓ 連線成功」才能繼續(防止填錯資料把 broker 卡死)
5. 按 **「儲存並啟用」** — 面板會自動連 broker 並推 HA discovery
6. 開 Home Assistant → **設定 → 裝置與服務 → MQTT** → 應看到新裝置

> 💡 **首次啟用後 5 秒內** HA 就能控制。如果一直看不到 entity,先檢查 broker IP / 帳密、HA 端 MQTT integration 是否正常運作。

### 6.3 HA 看得到的功能

**🔥 暖風機**

| HA Entity | 類型 | 控制 |
|---|---|---|
| `switch.*_heat` | `switch` | 暖房 開 / 關 |
| `switch.*_cool` | `switch` | 涼風 開 / 關 |
| `switch.*_vent` | `switch` | 換氣 開 / 關 |
| `switch.*_dry` | `switch` | 乾燥 開 / 關 |
| `sensor.*_roomtemp` | `sensor` | 室溫(唯讀)|

**🌬️ 全熱交換器**

| HA Entity | 類型 | 控制 |
|---|---|---|
| `fan.*_fan` | `fan` | 開關 + 風量 |
| `sensor.*_outtemp` | `sensor` | 外氣溫度(唯讀,視機型)|
| `sensor.*_rettemp` | `sensor` | 回風溫度(唯讀,視機型)|

**💧 米多力除濕機**

| HA Entity | 類型 | 控制 |
|---|---|---|
| `humidifier.*_dehumi` | `humidifier` | 除濕開關 + 目標濕度 |
| `fan.*_airpurifier` | `fan` | 清淨開關 + 風量 |
| `switch.*_ventilation` | `switch` | 換氣 / 送風 |
| `sensor.*_temperature` | `sensor` | 室溫(唯讀)|
| `binary_sensor.*_water` | `binary_sensor` | 水滿 / 漏水 |

在 HA 操作後,Apple Home 也會即時同步顯示(雙向同步)。

### 6.4 MQTT Topic Schema(進階)

```
automate/deltapanel/{mac6}/{entity}/state          ← 狀態 (JSON, retained)
automate/deltapanel/{mac6}/{entity}/set            ← 控制指令
automate/deltapanel/{mac6}/availability            ← LWT (online / offline)
homeassistant/{component}/{entity}_{mac6}/config   ← HA Auto-Discovery (retained)
```

- `{mac6}` = 面板 MAC 後 6 碼
- `{entity}` = `heat` / `cool` / `vent` / `dry` / `roomtemp` / `fan` / `dehumi` …(見 6.3)
- 米多力機型的前綴為 `automate/medole/{mac6}/…`

> 進階使用者可用 `mosquitto_sub` / Node-RED 等自製 dashboard,不一定要走 HA。

### 6.5 停用 / 移除

| 操作 | 結果 |
|---|---|
| **「停用」**按鈕 | 中斷 MQTT 連線,設定保留 — 之後可重新啟用 |
| **「刪除設定」**按鈕 | 清掉 broker IP / 帳密,HA 端 entity 也會被移除(retained config 清空)|
| **工廠重置** | 一併清掉 MQTT 設定(見[第 8 章](#8-工廠重置))|

> 💡 「停用」跟「刪除」都會主動通知 HA 把 entity 從清單拿掉(不只是顯示 unavailable),不會留下孤兒 entity。

<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">9. 故障排除</h2>

### 9.1 配對相關

| 現象 | 原因 | 處理 |
|---|---|---|
| 家庭 App 找不到裝置 | Wi-Fi 是 5 GHz / 距離太遠 | 走 2.4 GHz、靠近路由器 |
| 顯示「無法加入配件」 | 無 Home Hub / 配對碼錯 | 確認家中有 HomePod / Apple TV / 常駐 iPad;重掃面板 QR |
| 配對後顯示「無回應」 | 不同 SSID / mDNS 問題 | 手機與面板同一個 2.4 GHz SSID,關「訪客隔離」 |
| 換過機型後家庭 App 顯示異常 | 機型變更會改變裝置結構 | 先在家庭 App 移除裝置,再重新加入 |

### 9.2 運作相關

| 現象 | 原因 | 處理 |
|---|---|---|
| Apple Home 顯示離線 | Wi-Fi 斷線 / 路由器重啟 | 等 1–2 分鐘自動回連 |
| 面板操作但家電沒反應 | RS-485 通訊異常 / 家電關機 | 確認家電電源、RS-485 接線(A/B 是否接反)|
| 室溫 / 濕度顯示 `--` | 家電未回報 / 尚未輪詢 | 剛開機第一輪輪詢前會顯示 `--`;持續 `--` 請檢查 RS-485 |
| 螢幕內容位移 | 顯示異常 | 設定頁按「刷新」重建畫面 |

### 9.3 MQTT 相關

| 現象 | 原因 | 處理 |
|---|---|---|
| 連線測試失敗 | broker IP / 帳密錯、防火牆 | 確認 Mosquitto 執行中、port 1883 可通 |
| HA 看不到 entity | discovery 未推送 | 確認 HA 的 MQTT integration 正常;重按「儲存並啟用」 |
| 管道風機 / 阿拉斯加沒有 entity | 該機型尚未支援 MQTT | 正常現象,見 [6.1](#61-適用對象) |

### 9.4 OTA 相關

| 現象 | 原因 | 處理 |
|---|---|---|
| 版更顯示「無法取得版本」 | 連不到雲端 | 檢查網路 / DNS |
| 「開始更新」卡住 | 下載中斷 | 等 5 分鐘;不行重啟再試 |

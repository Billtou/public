<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">1. 產品簡介</h2>

LuxOne 是 AUTOMATE 推出的環境照度感測器,**直接接入 Apple Home / Google Home / Home Assistant / SmartThings / Alexa**,不需專屬 Hub。

裝置內建 **BH1750 光照度感測 IC**(1 lx 解析度,0–65535 lx 量程),即時把環境亮度推給 Matter 智能生態,讓你可以依「天亮 / 天黑 / 自然光充足」這類條件做自動化:燈具開關、窗簾升降、戶外燈節能、安全照明等。

### 2.1 安裝位置建議

LuxOne 的 BH1750 感測頭朝**機身上方**(或頂部圓孔位置),建議:

| 場景 | 建議擺放 |
|---|---|
| **客廳 / 書房** | 桌面平放,離窗戶 1–2 m,反映你日常使用區域的亮度 |
| **臥室** | 床頭櫃上,避免被檯燈直射,反映自然光 |
| **戶外 / 露台** | 用防水盒,朝天空但避雨,反映 EVI 環境光 |
| **植物角** | 跟植物同高度同朝向,反映植物實際接收光照 |
| **天花板感測** | 朝下安裝倒掛,反映地面工作面亮度(類似 PIR 安裝法)|

<blockquote style="border-left:4px solid #ff6f48;background:#fff7f4;padding:14px 18px;margin:16px 0;border-radius:0 12px 12px 0;color:#5a3520">
💡 避免:陽光直射(會飽和到 65535)、靠近燈具(局部過亮)、被遮蔽(讀值總是偏低)。
</blockquote>

### 3.1 按鍵操作

機身有一顆多功能配對鍵:

| 操作 | 結果 |
|---|---|
| **單擊**(< 0.6 秒)| 切換**蜂鳴器靜音** |
| **雙擊**(0.6 秒內連按)| 切換**待機 LED 指示燈**(夜間想關掉 LED 光污染就雙擊)|
| 持續長按 ≈ 5 秒 | 進入重置警告(LED 快閃提示)|
| 持續長按 ≥ 10 秒 | **工廠重置觸發**,清除 Matter 配對與 Wi-Fi,自動重啟 |

### 3.2 狀態 LED

| LED 行為 | 含意 |
|---|---|
| **2 Hz 閃爍**(快速)| 開機初始化 |
| **1 Hz 閃爍**(每秒一下)| **等待配對** — 還沒被任何生態加入 |
| **常亮 / 熄滅**(取決於雙擊設定)| 已配對,正常運作中 |
| **10 Hz 閃爍**(極快)| 長按重置中(放開可取消;繼續壓到 10 秒會觸發重置)|

### 3.3 蜂鳴器提示

| 事件 | 提示音 |
|---|---|
| 開機完成 | 三聲漸高音 |
| 配對成功 | 兩聲漸高音 |
| 工廠重置完成 | 一聲長嗶 |
| 韌體更新中 | 持續短嗶,完成後三聲漸高 |

單擊配對鍵可全域靜音蜂鳴器。

<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">5. 跨生態接入(Matter Multi-Admin)</h2>

### 5.1 機制說明

Matter 規範允許**同一台 LuxOne 同時加入多個生態**,最多 **5 個 Fabric**。例如同時被 Apple Home 與 Home Assistant 控制,lux 讀值會同步出現在兩個 App,可分別寫自動化規則。

<blockquote style="border-left:4px solid #ff6f48;background:#fff7f4;padding:14px 18px;margin:16px 0;border-radius:0 12px 12px 0;color:#5a3520">
⚠️ <b>關鍵原則</b>:第二個生態起,<b>不能用出廠那組</b> <code>3642-630-6820</code> — 必須由<b>第一個已配對的生態</b>產生一組「分享配對碼」(每次新的、有時效)。
</blockquote>

### 5.2 從 Apple Home 分享給其他生態

1. 家庭 App → 長按 LuxOne → **設定**(齒輪圖示)
2. 滑到底,找 **「開啟配對模式」**
3. App 顯示一組 11 碼**拷貝代碼**(15 分鐘內有效)

### 7.3 為什麼是 lux 而不是「亮/暗」

LuxOne 給出**真實連續數值**,自動化可以做「漸進式判斷」:
- 不會在「lux=99 開燈 / lux=101 關燈」反覆抖動
- 可以用「平均值」「移動視窗」「持續時間」這類進階規則(HA / Home Assistant 自動化)
- 上游 controller 可看趨勢做決策(例:lux 持續下降才判定天黑,避免雲飄過誤觸發)

### 7.4 跟其他感測器組合

| 組合 | 用途 |
|---|---|
| LuxOne + MotionOne | 「人在 + 暗」→ 開燈;「人在 + 亮」→ 不動 |
| LuxOne + AirCube | 一台空品 + 一台照度,寫「白天通風,晚上靜音」 |
| LuxOne + AS-P3(Pro) | AS-P3 內建照度但可外加一台 LuxOne 放窗邊,反映室外實際亮度 |

<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">10. 故障排除</h2>

### 10.1 配對相關

| 現象 | 原因 | 處理 |
|---|---|---|
| App 找不到裝置 | 沒進入配對模式 / 手機藍牙關閉 / 距離太遠 | 確認手機藍牙開、距離 < 1 m、Wi-Fi 是 2.4 GHz |
| App 顯示「無法加入」| 配對碼錯誤 / Fabric 已滿 | 重新輸入碼;若已用 5 個生態,先移除其一 |
| 第二個生態加不進 | 用了出廠那組碼 | 改用第一個生態的「分享碼」(章節 5)|
| 配對成功但 App 顯示離線 | DNS / mDNS 問題 / Wi-Fi 沒共用網段 | 確認手機跟 LuxOne 同一個 2.4G SSID,路由器不要走 Guest 隔離 |

### 10.2 lux 讀值相關

| 現象 | 原因 | 處理 |
|---|---|---|
| 讀值永遠 0 lx | BH1750 通訊失敗 / 感測頭被完全遮蔽 | 重新插拔電源;確認感測頭朝外無遮蔽 |
| 讀值永遠 65535 lx(飽和)| 陽光直射 / 強燈直射 | 移到沒有直接強光的位置;BH1750 量程上限 65535 |
| 讀值跳動很大 | 旁邊有閃爍光源(螢光燈、LED PWM)| 視為正常 — BH1750 對 PWM 光源敏感;3 秒取樣已盡力平均 |
| 讀值跟其他 lux 計差很多 | 角度、遮蔽不同 / 感測元件容差 ±20% | 用 Web UI 確認讀值穩定即可,絕對精度不適合做光學測量 |
| HA 看到 illuminance 但 Apple Home 沒值 | iOS 版本太舊 | 升級到 iOS 16.1+(Matter 1.0 IlluminanceMeasurement support)|

### 10.3 通訊相關

| 現象 | 原因 | 處理 |
|---|---|---|
| App 中 LuxOne 顯示離線 | Wi-Fi 斷線 / 路由器重啟 | 等 1–2 分鐘自動回連;不行就重新通電 |
| Web UI 進不去 | mDNS 沒響應 / 跨網段 | 試 IP 直連;確認手機在同一個 SSID |

### 10.4 OTA 相關

| 現象 | 原因 | 處理 |
|---|---|---|
| 版更頁顯示「無法取得版本」| 裝置連不到 GitHub | 檢查網路,可能是 DNS / 網路 / 防火牆問題 |
| 「開始更新」後一直 0% | 下載卡住 | 等 5 分鐘;不行就重啟裝置再試 |
| 更新失敗 | 韌體損毀 / 供電不足 | 重啟裝置(會走舊版),確認供電後重試 |

- 🌞 避免陽光長期直射機身,塑膠殼長期高溫可能變形

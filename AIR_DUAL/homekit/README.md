<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">1. 產品簡介</h2>

AirDual 是 AUTOMATE 推出的 HomeKit 智能除濕清淨二合一控制器,**直接接入 Apple Home**,不需專屬 Hub 或雲端帳號。

二合一機本體沿用 TaiSEIA 標準通訊協定,模組透過 UART 跟原廠 MCU 通訊。Apple Home 上會看到**兩台子裝置**:

<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">3. 指示燈與按鍵行為</h2>

### 3.1 狀態 LED

模組上單色藍燈,依**優先序**顯示連線狀態:

| 條件 | LED 行為 | 意義 |
|---|---|---|
| 重置警告(長按 ≥ 5s)| **急閃** 100ms 亮 / 100ms 滅 | 「再按 5 秒會觸發工廠重置」警告 |
| Wi-Fi 未連線 | **慢閃** 0.2s 亮 / 1.8s 滅 | 配網模式 / Wi-Fi 重連中 |
| HomeKit 未配對 / UART 未連 | **雙閃** 2 秒內閃兩下 | Wi-Fi 通了但 HomeKit 還沒加入 |
| 正常運轉 | **熄滅** | 三條都通,不擾人 |

<blockquote style="border-left:4px solid #007AFF;background:#eff7ff;padding:14px 18px;margin:16px 0;border-radius:0 12px 12px 0;color:#1c3d5a">
💡 規範:LED 預設不打擾。閒置正常時不亮,有問題才亮。
</blockquote>

### 3.2 重置按鈕

| 按法 | 時間 | 結果 |
|---|---|---|
| 短按 | < 0.6 秒 | 忽略(debounce)|
| 單擊 | 0.6 – 5 秒 | 切換主電源 |
| 長按 | 5 – 10 秒 | 重置警告(LED 急閃)|
| 長按 | ≥ 10 秒 | **工廠重置** |

### 3.3 原廠面板 / 遙控

走廠商既有通路,**獨立於 AirDual 模組**:
- 不受 Wi-Fi 斷線影響
- 操作的狀態變化,Apple Home 端 1 秒內同步看到
- 反之亦然(Apple Home 控制 → 原廠面板 LCD 也會同步)

<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">6. Home Assistant 整合 (MQTT)</h2>

本機內建 MQTT bridge,**Apple Home 與 Home Assistant 可同時使用**(雙生態,狀態雙向同步)。
適合已有 Mosquitto / HA 環境的進階使用者。

### 6.1 適用對象

| 你是 | 用不用得到 |
|---|---|
| 只用 Apple Home,沒 HA | 跳過本章,Apple Home 功能完整不缺 |
| 已有 Home Assistant + Mosquitto broker | 跟著本章設定,可在 HA 看到對應 entity |
| 沒裝 Mosquitto | 先在 HA 裝 `Mosquitto broker` add-on,設帳密再回來 |

### 6.2 設定步驟

1. 用瀏覽器打開 **`http://<裝置-IP>/engineer`**(裝置 IP 從家庭 App → 點裝置 → 設定 → 看 IP,或在區網內以 `http://automate-airdual-XXXXXX.local` 連入)
2. 頁面捲到 **「MQTT / Home Assistant」** 卡片
3. 填入:
   | 欄位 | 範例 | 說明 |
   |---|---|---|
   | Broker IP | `192.168.1.10` | Mosquitto 主機 IP |
   | Port | `1883` | 預設 1883 |
   | Username | `mqtt-user` | broker 帳號 |
   | Password | `********` | broker 密碼 |
4. 按 **「連線測試」** — 必須看到綠色「✓ 連線成功」才能繼續(防止填錯誤資料把 broker 卡死)
5. 按 **「儲存並啟用」** — 裝置會自動連 broker 並推 HA discovery
6. 開 Home Assistant → **設定 → 裝置與服務 → MQTT** → 應看到新裝置 **「automate-airdual-XXXXXX」**(XXXXXX = MAC 後 6 碼)

> 💡 **首次啟用後 5 秒內** HA 就能控制設備。如果一直看不到 entity,先檢查 broker IP / 帳密、HA 端 MQTT integration 是否正常運作。

### 6.4 MQTT Topic Schema(進階)

```
automate/airdual/{mac6}/dehumi/state              ← 除濕機 state(JSON,retained)
automate/airdual/{mac6}/dehumi/set                ← ON / OFF
automate/airdual/{mac6}/dehumi/mode/set           ← auto / normal / continuous / clothes_dry / air
automate/airdual/{mac6}/dehumi/humidity/set       ← 30 ~ 70(目標濕度)
automate/airdual/{mac6}/airpurifier/state         ← 空清 state(只在二合一機型)
automate/airdual/{mac6}/airpurifier/set           ← ON / OFF
automate/airdual/{mac6}/airpurifier/percentage/set ← 1 ~ 100(風量)
automate/airdual/{mac6}/availability              ← LWT (online / offline)
homeassistant/{humidifier|fan}/airdual_{mac6}[_suffix]/config  ← HA Auto-Discovery
```

ERV 機型 topic 不帶 suffix:
```
automate/airdual/{mac6}/state
automate/airdual/{mac6}/set                ← ON / OFF
automate/airdual/{mac6}/percentage/set     ← 風量
automate/airdual/{mac6}/preset_mode/set    ← Auto / HeatExchange / Bypass / Recirculate / Purify
```

> 進階使用者可用 `mosquitto_sub` / Node-RED 等自製 dashboard,不一定要走 HA。

### 6.5 停用 / 移除

| 操作 | 結果 |
|---|---|
| **「停用」**按鈕 | 中斷 MQTT 連線,設定保留 — 之後可重新啟用 |
| **「刪除設定」**按鈕 | 清掉 broker IP / 帳密,HA 端 entity 也會被移除(retained config 清空)|
| **工廠重置** | 一併清掉 MQTT 設定(見 [第 10 章](#10-工廠重置))|

> 💡 「停用」跟「刪除」都會主動通知 HA 把 entity 從清單拿掉(不只是顯示 unavailable),不會留下孤兒 entity。

<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">8. 模式與功能</h2>

### 8.1 運轉模式

由 Apple Home 或原廠面板切換(Web UI 不提供模式控制):

| 模式 | 用途 |
|---|---|
| **自動除濕** | 機台依當前濕度自動決定壓縮機 ON/OFF,維持在目標濕度 |
| **保持乾燥** | 比自動更積極,維持較低濕度(衣櫃 / 倉庫場景)|
| **連續除濕** | 不看目標濕度,壓縮機持續運轉至水箱滿 |
| **衣物乾燥** | 風扇強勁 + 持續除濕,適合室內曬衣 |
| **空氣清淨** | 壓縮機停,只跑風扇 + 濾網 + 離子,純空氣淨化 |
| **防霉抑菌** | 機台依「防霉演算法」自動間歇除濕 + 開離子 |

### 8.2 風量段位

6 段:**自動 / 靜音 / 微風 / 標準 / 強風 / 急速**。Apple Home 風扇磚塊滑桿任意中間值會自動量化到最近段位。

### 8.3 nanoeX 離子

**獨立硬體開關**,不依 mode 自動切換:
- 打開後:離子發生器持續運轉,不論機台是除濕、空清、防霉、衣物乾燥都會跑
- 關閉後:離子發生器停,即使切到防霉 mode 也不會自動開
- 整機 OFF:離子物理上停(沒電),但 state 記得;再開機通常自動恢復

### 8.4 水箱與濾網

| 顯示 | 處理 |
|---|---|
| 水箱「正常」 | 不用管 |
| 水箱「**已滿**」(紅 pill)| 倒水後重新插入,機台自動繼續運轉 |
| 濾網「正常」 | 不用管 |
| 濾網「**需更換**」(橘 pill)| 拆下清洗 / 更換濾網。完成後依機台說明書按重置鍵清除提示 |

<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">11. 故障排除</h2>

### 11.1 配對相關

| 現象 | 原因 | 處理 |
|---|---|---|
| 家庭 App 找不到裝置 | 距離太遠 / Wi-Fi 是 5 GHz | 確認距離 < 1 m、走 2.4 GHz |
| 顯示「無法加入配件」 | 無 Home Hub / 配對碼錯 | 確認家中有 HomePod / Apple TV / iPad;重掃 QR |
| 配對中途斷開 | 路由器訊號弱 / 模組電源不穩 | 移近路由器;檢查 AC 電源 |
| 配對後顯示「無回應」 | DNS / mDNS / 不同 SSID | 同一個 2.4 GHz SSID,關「Guest 隔離」 |

### 11.2 運作相關

| 現象 | 原因 | 處理 |
|---|---|---|
| Apple Home 顯示離線 | Wi-Fi 斷線 / 路由器重啟 | 等 1–2 分鐘自動回連 |
| 拉滑桿但機台沒反應 | UART 通訊異常 / 機台關機 | 確認機台電源 ON;重啟模組 |
| Web UI 數值顯示 `—` | 機台沒回報該感測值 | 部分機型不支援、或剛開機 polling 還沒一輪 |
| 原廠面板可動,Apple Home 沒同步 | UART 通訊異常 | 重啟模組 / 等 1 秒 polling 同步 |

### 11.3 感測讀數異常

| 現象 | 原因 | 處理 |
|---|---|---|
| 室溫顯示「-- °C」 | 感測器離線 / UART 沒通 | 重啟機台 + 模組 |
| PM2.5 一直 `—` | 機型不支援 / Profile 設錯 | 確認包裝盒側標籤 |
| 空氣品質燈號永遠 `—` | 機型只有除濕、無空清 | Apple Home 不會出現燈號(預期行為)|

### 11.4 OTA 相關

| 現象 | 原因 | 處理 |
|---|---|---|
| 版更顯示「**無法取得版本**」 | 連不到 GitHub | 檢查網路 / DNS |
| 「開始更新」一直 0% | 下載卡住 | 等 5 分鐘;不行重啟再試 |
| 更新失敗 | 韌體損毀 / 供電不足 | 重啟(會走舊版),確認供電後重試 |

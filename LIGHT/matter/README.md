<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">4. 接入智能生態(初次配對)</h2>

### 4.1 配對前確認

- [ ] 路由器 **Wi-Fi 2.4 GHz** 可用(LTC3 不支援 5 GHz)
- [ ] 手機 **藍牙開啟**,跟 LT 距離 < 1 公尺
- [ ] 手機**已加入** Matter 平台(Apple Home / Google Home / Home Assistant / Alexa)
- [ ] 至少有一台 Matter Controller 在線:
  - **Apple**:iPhone / iPad / HomePod / Apple TV(iOS 16.1+,**自適應照明需 iOS 18+ 並建議 iOS 26+**)
  - **Google**:Nest Hub / Android 14+ 手機
  - **Home Assistant**:主機 / iPhone / Android 14+ 手機
  - **Alexa**:Echo Hub / Echo (4th gen) / Echo Show

### 4.2 Apple Home

1. 通電 LT,等裝置完成上電(LED 燈具會閃爍一下進入配對模式)
2. iPhone 開「家庭」App → 右上角 **+** → **加入配件**
3. 對準 QR Code 掃描;或選「沒有代碼/無法掃描」→ 輸入 `3642-630-6820`
4. iPhone 自動傳 Wi-Fi 憑證
5. 配對完成 → 出現「燈具 1」/「燈具 2」(CW)或「LT-RGBW」/「LT-RGBCW」(單 EP 機型)

### 4.3 Home Assistant

1. 通電 LT,等裝置進入配對模式
2. 開 Home Assistant App → 設定 → Matter → 新增裝置 → **全新的裝置**
3. 掃 QR 或輸入 `3642-630-6820`
4. App 自動完成

### 4.4 Google Home

1. 通電 LT
2. 開「Google Home」App → 右上 **+** → **設定裝置** → **新裝置** → 選擇家庭
3. App 自動偵測「Matter 裝置」→ 點擊 → 掃 QR 或輸入 `3642-630-6820`
4. 依 App 指示完成 Wi-Fi 設定
5. 配對完成,可在「燈光」分類找到

### 4.5 Amazon Alexa

1. 通電 LT
2. 開 Alexa App → 裝置 → **+** → **新增裝置** → **其他** → **Matter**
3. 掃 QR 或輸入 `3642-630-6820`
4. 完成

<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">5. 跨生態接入(Matter Multi-Admin)</h2>

### 5.1 機制說明

Matter 規範允許**同一台 LT 同時加入多個生態**,最多 **5 個 Fabric**。例如同時被 Apple Home 與 Home Assistant 控制,燈具狀態會同步出現在兩個 App。

<blockquote style="border-left:4px solid #ff6f48;background:#fff7f4;padding:14px 18px;margin:16px 0;border-radius:0 12px 12px 0;color:#5a3520">
⚠️ <b>關鍵原則</b>:第二個生態起,<b>不能用出廠那組 <code>3642-630-6820</code></b> — 必須由<b>第一個已配對的生態</b>產生一組「分享配對碼」(每次新的、有時效)。
</blockquote>

### 5.2 從 Apple Home 分享給其他生態

1. 家庭 App → 長按燈具 → **設定**(齒輪圖示)
2. 滑到底,找 **「開啟配對模式」**
3. App 顯示一組 11 碼**拷貝代碼**(15 分鐘內有效)
4. 在第二個生態 App「加入配件」流程輸入這組新碼

### 5.3 從 Home Assistant 分享給其他生態

1. Home Assistant App → 設定 → Matter → 指定轉出燈具 → **複製代碼 / 分享裝置**
2. **複製代碼**
3. 在第二個生態 App「加入配件」流程輸入這組新碼

### 5.4 常見誤區

| 誤操作 | 後果 | 正確做法 |
|---|---|---|
| 第二個生態用出廠 `3642-630-6820` | 配對失敗 | 用第一個生態生出來的「分享碼」 |
| 用 Apple 重置裝置後再加入 Google | Apple 那邊也會掉 | 不要重置,用「Multi-Admin 分享」流程 |
| 已加入 5 個生態還想加 | 失敗 | 移除其中一個再加 |
| Wi-Fi 不通就嘗試多 admin | 失敗 | 先確認原配對生態正常,Wi-Fi 通才能分享 |

<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">6. Web UI:首頁(日常設定)</h2>

### 6.1 連線方式

配對成功後,LT 在區網提供 Web UI。打開瀏覽器,網址兩種擇一:

- **直接 IP**:在生態 App 內查看裝置 IP(例 `http://192.168.1.123`)
- **主機名稱**(依機型):
  - CW:`http://automate-ltcw-XXXXXX.local`
  - RGBW:`http://automate-ltrgbw-XXXXXX.local`
  - RGBCW:`http://automate-ltrgbcw-XXXXXX.local`

> 手機與 LT 必須在**同一個 Wi-Fi 網段**。

### 6.2 裝置資訊與即時狀態

| 區塊 | 內容 |
|---|---|
| **裝置資訊** | 韌體版本、可用記憶體、運行時間、連線狀態 |
| **即時狀態** | 各 EP 的 ON/OFF、亮度、色溫(mireds)即時值 |

### 6.3 自適應燈光偏好(色溫保護)

> 即時生效,不需重啟、不需重新配對

限制 HomeKit 自適應照明 / 手動拉桿不會比指定色溫更暖,適合**不喜歡偏黃光**的環境。

| 選項 | 行為 |
|------|------|
| **不限制**(預設)| LED 完全跟隨上層指令(可達物理範圍最暖端)|
| **3000K** | 不會比柔白更暖 |
| **3500K** | 不會比溫白更暖 |
| **3750K** | 不會比亮白更暖 |
| **4000K** | 不會比中性白更暖 |

> 此設定僅在 firmware 端做 clamp,**不影響** Apple Home / HA 滑桿可調範圍 — 即使滑桿拉到最暖,實際 LED 輸出也不會超過你選的限制。

### 6.4 計時器與連動設定

| 設定 | 說明 |
|------|------|
| **來電狀態** | 通電後燈具預設狀態(開燈/關燈/同上次狀態)|
| **MotionEX 延遲關燈分鐘** | 可選 1 / 3 / 5 / 10 / 15 分鐘 |
| **燈具 1 連動 MotionEX** | 是否讓 EP1 隨外接 MotionEX 感測器自動開關 |
| **燈具 2 連動 MotionEX**(僅 CW)| EP2 同上 |

> 關閉「連動」後,MotionEX 訊號完全不影響燈具狀態;計時器到期也不會關燈。

<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">7. HomeKit 自適應照明</h2>

### 7.1 系統需求

依 Apple 官方[ Adaptive Lighting over Matter R1 規格](https://developer.apple.com/apple-home/downloads/Adaptive-Lighting-over-Matter-Accessory-Requirements.pdf):

- iPhone / iPad **iOS 18+**(建議 **iOS 26+** 自動啟用)
- 燈具支援色溫範圍至少 **2700K – 5000K**(LTC3 系列預設 2700K-6000K,符合)
- 燈具支援 **Matter 1.4+**(LT v2.0+ 已支援)
- Apple Home Hub(HomePod / Apple TV / iPad)在線

### 7.2 家庭地址設定

自適應照明根據**家庭地址**算當地日出/日落時間。地址設定:

1. iPhone 開「**地圖**」App
2. 下方「喜愛」→ **+ 加入** → 輸入住家地址
3. 標籤選 **住家**,完成
4. 確認 iPhone「設定 → 隱私權與安全性 → 定位服務 → 系統服務 → 設定與時區」**已開啟**

### 7.3 曲線運作原理

```
[Apple Home Hub] ──每 24 小時計算曲線──→ [LT 燈具]
       ↑
  讀取家庭地址
  → 算當地日出/日落
  → 套用 Apple 內建 circadian 曲線
  → 推送 24 小時時間↔CT 對照表
```

**曲線特性**:

- 日出後 ~30 分:開始從暖端慢慢升溫
- **太陽上中天**(solar noon,非時鐘 12:00):最冷峰
- 日落前 ~1-2 小時:開始回暖
- 日落後 ~30 分:完全回到暖光
- 夜間至日出前:維持 2700K 暖光平台

### 7.4 搭配「色溫保護」

如果你覺得 Apple 自適應曲線晚上太暖,可以用 Web UI 首頁的「**自適應燈光偏好**」(章節 6.3)做軟限制:

- 例:設定「**不更暖於 3500K**」→ 即使 Apple 自適應送 2700K,實際 LED 也只到 3500K

<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">8. OTA 韌體更新</h2>

### 8.1 更新流程

1. 連線 Web UI → 底部 **版更** Tab
2. 系統自動向官方檢查最新版本
3. 若有新版,顯示「有新版本可更新」+ 「開始更新」按鈕
4. 點擊「開始更新」→ 確認對話框 → 自動下載
5. 下載完成自動重啟,Web UI 自動 reload 回首頁

整個過程約 1–2 分鐘,期間裝置會暫時離線。

### 8.2 更新會保留 / 清除什麼

| 項目 | 是否保留 |
|---|---|
| Matter 配對(所有生態)| ✅ 保留 |
| Wi-Fi 帳密 | ✅ 保留 |
| 自適應燈光偏好(色溫保護)| ✅ 保留 |
| 計時器與連動設定 | ✅ 保留 |
| 來電狀態 | ✅ 保留 |
| 硬體色溫範圍 | ✅ 保留 |
| 燈具 Profile | ✅ 保留 |
| 韌體版本 | ❌ 換新版 |

<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">9. 工廠重置</h2>

### 9.1 硬體重置

- 連續快速開關 AC 開關 5 次以上,進入重置警告
- 持續長按 ≥ 10 秒(實作為連續開關)→ **工廠重置觸發**
- 自動重啟,進入配對模式

### 9.2 Web UI 重置

- 版更頁 → 「裝置資料清除」card → 「清除 NVS(出廠重置)」
- 確認對話框 → 自動重啟

### 9.3 重置會清除什麼

| 項目 | 是否清除 |
|---|---|
| Matter 配對(所有生態)| ✅ 清除 |
| Wi-Fi 帳密 | ✅ 清除 |
| 自適應燈光偏好 / 計時器設定 / 來電狀態 | ✅ 重置為預設 |
| 硬體色溫範圍 | ✅ 重置為 2700K-6000K |
| 燈具 Profile | ❌ 不變(保留出廠選擇)|
| 韌體版本 | ❌ 不變 |
| 裝置認證資料 / DAC | ❌ 不變(內建出廠憑證,不會掉)|

> 重置後跟全新開封一樣,需重新配對。

<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">10. 故障排除</h2>

### 10.1 配對相關

| 現象 | 原因 | 處理 |
|---|---|---|
| App 找不到裝置 | 沒進入配對模式 / 手機藍牙關閉 / 距離太遠 | 確認手機藍牙開、距離 < 1 m、Wi-Fi 是 2.4 GHz |
| App 顯示「無法加入」| 配對碼錯誤 / Fabric 已滿 | 重新輸入碼;若已用 5 個生態,先移除其一 |
| 第二個生態加不進 | 用了出廠那組碼 | 改用第一個生態的「分享碼」(章節 5)|

### 10.2 HomeKit 配對失敗

| 現象 | 原因 | 處理 |
|---|---|---|
| **「無法加入配件 Pairing failed」** | iPhone 連的是 5GHz,LT 上不了該 AP | 換到 2.4GHz SSID 再配對 |
| Apple Home 失敗但 HA 可加入 | iCloud / HomePod 緩存舊 fabric | 重啟 iPhone + HomePod / Apple TV;不行就重置網路設定 |
| 配對中途斷開 | 供電不足造成裝置重啟 | 確認 LT 有穩定 AC 電源 |
| 配對後 HomeKit 看不到「自適應照明」開關 | iOS 版本太舊 / 硬體色溫範圍不符 | 升級 iOS 到 18+;確認 Web UI 工程模式色溫範圍涵蓋 2700K-5000K |

### 10.3 自適應照明異常

| 現象 | 原因 | 處理 |
|---|---|---|
| 中午沒到冷白 | HomeKit 緩存舊範圍 / Hub 沒重抓地址 | 移除配件 → 重啟 HomePod → 重新配對 |
| 整天偏暖 / 變化時間錯 | 家庭地址沒設 / 時區錯 | 地圖 App 重設家庭地址;確認 iPhone / Hub 時區自動 |
| 變化「卡卡的」一階一階 | Hub 推送間隔 ~10-60 分鐘 | Apple 設計如此,無解;搭配 LT 的「色溫保護」可緩解 |
| 整體偏黃,想限制 | 預期行為 | 首頁「自適應燈光偏好」設「不更暖於 3500K」之類 |

### 10.4 運作相關

| 現象 | 原因 | 處理 |
|---|---|---|
| App 中燈具顯示離線 | Wi-Fi 斷線 / 路由器重啟 | 等 1–2 分鐘自動回連;不行就重新通電 |
| 燈具一直開或一直關 | 連動 MotionEX 卡死 | Web UI 關閉 MotionEX 連動 toggle |
| 漸變不順 / 抖動 | 供電不穩 | 確認 AC 電源穩定,不要走太細的延長線 |
| 色溫拉到最暖卻變更冷 | 啟用了「色溫保護」軟限制 | Web UI 首頁將「最低色溫保護」改為「不限制」|

### 10.5 OTA 相關

| 現象 | 原因 | 處理 |
|---|---|---|
| 版更頁顯示「無法取得版本」| 裝置連不到 GitHub | 檢查網路,可能是 DNS / 網路 / 防火牆問題 |
| 「開始更新」後一直 0% | 下載卡住 | 等 5 分鐘;不行就重啟裝置再試 |
| 更新失敗 | 韌體損毀 / 供電不足 | 重啟裝置(會走舊版),確認供電後重試 |

<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">11. 安全使用</h2>

<blockquote style="border-left:4px solid #ff6f48;background:#fff7f4;padding:14px 18px;margin:16px 0;border-radius:0 12px 12px 0;color:#5a3520">
⚠️ <b>AC 110V/220V 高壓側請務必由合格電工安裝</b>,觸電會致命。接線前先斷電,確認電壓符合機型。
</blockquote>

- 接線前**先斷電**,確認電壓符合機型(110V / 220V 不同)
- 燈具 LED 輸出側勿短路或反接(共陽極正極為 +12V / +24V)
- 工作環境溫度 -10–50 °C,**勿放置於暖氣 / 廚房灶台旁**
- 戶外使用請另購防水殼
- 請勿拆殼 / 改裝 — 內含高壓元件與無線電路,改裝會違反電信法規

<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">5. 跨生態接入(Matter Multi-Admin)</h2>

### 5.1 機制說明

Matter 規範允許**同一台 AS-P3 同時加入多個生態**,最多 **5 個 Fabric**。例如同時被 Apple Home 與 Home Assistant 控制,狀態會同步出現在兩個 App(雷達感測到人 → 兩邊都收到事件,可分別設自動化規則)。

<blockquote style="border-left:4px solid #ff6f48;background:#fff7f4;padding:14px 18px;margin:16px 0;border-radius:0 12px 12px 0;color:#5a3520">
⚠️ <b>關鍵原則</b>:第二個生態起,<b>不能用出廠那組</b> <code>3642-630-6820</code> — 必須由<b>第一個已配對的生態</b>產生一組「分享配對碼」(每次新的、有時效)。
</blockquote>

### 5.2 從 Apple Home 分享給其他生態

1. 家庭 App → 長按 AutoSpace → **設定**(齒輪圖示)
2. 滑到底,找 **「開啟配對模式」**
3. App 顯示一組 11 碼**拷貝代碼**(15 分鐘內有效)
4. 在第二個生態 App「加入配件」流程輸入這組新碼

### 5.3 從 Home Assistant 分享給其他生態

1. Home Assistant App → 設定 → Matter → 指定轉出裝置 → **複製代碼 / 分享裝置**
2. **複製代碼**
3. 在第二個生態 App「加入配件」流程輸入這組新碼

### 5.4 常見誤區

| 誤操作 | 後果 | 正確做法 |
|---|---|---|
| 第二個生態用出廠 `3642-630-6820` | 配對失敗 | 用第一個生態生出來的「分享碼」 |
| 用 Apple 重置裝置後再加入 Google | Apple 那邊也會掉 | 不要重置,用「Multi-Admin 分享」流程 |
| 已加入 5 個生態還想加 | 失敗 | 移除其中一個再加 |
| Wi-Fi 不通就嘗試多 admin | 失敗 | 先確認原配對生態正常,Wi-Fi 通才能分享 |

<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">7. 自動化邏輯與感測說明</h2>

### 7.1 主感測:24 GHz 毫米波雷達

跟傳統 PIR 相比,**mmWave 雷達**的優勢:

| 情境 | PIR | mmWave 雷達 |
|---|---|---|
| 大幅動作走路 | ✅ 觸發 | ✅ 觸發 |
| 坐著滑手機 | ❌ 漏判 | ✅ 觸發(微胸腹起伏)|
| 上廁所靜坐看手機 | ❌ 漏判(燈會關)| ✅ 持續觸發 |
| 隔玻璃窗 / 薄牆 | ❌ 部分穿透 | ✅ 部分穿透(需測試)|
| 受日光 / 熱源干擾 | ❌ 易假觸發 | ✅ 不受影響 |
| 反應速度 | 1–2 秒 | 0.5–1 秒 |
| 偵測距離(<8 m)| 4–6 m | 8 m |

**P1 / P2 機型額外加 PIR 是為了「冗餘確認」**:雷達 + PIR 同時觸發 = 高信心(避免雷達假觸發),其中一邊觸發 = 低信心(可選擇較短亮燈時間)。

### 7.2 自動化決策邏輯

預設規則(可在 Web UI 調整):

```
偵測到人(雷達 OR PIR)
  → IF (環境照度 < 閾值)
      → 開電燈
  → ELSE
      → 不動作(白天 / 已有自然光)

無人持續 N 秒(預設 2 分鐘)
  → 關電燈
  → IF (Pro 機型且當前 Mute 模式未啟動)
      → 蜂鳴器一聲短嗶(可選,提醒「我幫你關燈了」)

(Pro 機型,獨立規則)
濕度 > 閾值(預設 80%RH)
  → 開風扇換氣
濕度回落 < 閾值
  → 風扇繼續轉 N 分鐘(預設 15 分鐘)後關
```

### 7.3 手動覆蓋

任何時候**透過 Apple Home / HA / Web UI / 物理開關手動切換電燈或風扇**,自動化會**暫停 5 分鐘**(讓你「正在洗澡所以開燈」的手動操作不會被自動化關掉)。5 分鐘後恢復自動化判斷。

<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">10. 故障排除</h2>

### 10.1 配對相關

| 現象 | 原因 | 處理 |
|---|---|---|
| App 找不到裝置 | 沒進入配對模式 / 手機藍牙關閉 / 距離太遠 | 確認手機藍牙開、距離 < 1 m、Wi-Fi 是 2.4 GHz |
| App 顯示「無法加入」| 配對碼錯誤 / Fabric 已滿 | 重新輸入碼;若已用 5 個生態,先移除其一 |
| 第二個生態加不進 | 用了出廠那組碼 | 改用第一個生態的「分享碼」(章節 5)|
| 配對成功但 App 顯示離線 | DNS / mDNS 問題 / Wi-Fi 沒共用網段 | 確認手機跟 AS-P3 同一個 2.4G SSID,路由器不要走 Guest 隔離 |

### 10.2 感測 / 自動化相關

| 現象 | 原因 | 處理 |
|---|---|---|
| 雷達一直顯示「無人」 | 安裝位置偏 / 雷達朝錯方向 | 確認 AS-P3 朝下且覆蓋常人經過動線 |
| 雷達假觸發(沒人也亮)| 旁邊有風扇 / 窗簾擺動 / 雷達覆蓋到通風口 | 調整安裝角度遠離擺動物;P1/P2 機型可看 PIR 同時觸發才信 |
| 環境很亮但燈還是開 | 照度閾值設太高 | Web UI 調低「照度閾值」 |
| 環境很暗但燈不開 | 自動化關掉 / 照度閾值設太低 | 確認「自動化開關」是 ON,照度閾值 > 當前環境 lux |
| 走出去燈馬上關 | 「無人關燈延遲」太短 | Web UI 調長(浴室建議 5 min)|
| 浴室風扇沒啟動(Pro)| 濕度閾值設太高 / 自動化關閉 | 確認自動化 ON,濕度閾值 ≤ 80%RH |
| 風扇一直開不關 | 濕度感測器壞 / 環境真的高濕 | 開 Web UI 看當前濕度;> 95% 持續可能感測器卡水珠 |

### 10.3 通訊相關

| 現象 | 原因 | 處理 |
|---|---|---|
| App 中 AS-P3 顯示離線 | Wi-Fi 斷線 / 路由器重啟 | 等 1–2 分鐘自動回連;不行就重新通電 |
| Apple Home 開關沒反應 | Wi-Fi 不通 / Matter session 斷 | 確認 Wi-Fi 通;不行就重啟 AS-P3 AC |
| 雷達狀態變化 App 看不到 | 推播被路由器擋掉 | 確認路由器 mDNS 跟 multicast 沒關 |

### 10.4 OTA 相關

| 現象 | 原因 | 處理 |
|---|---|---|
| 版更頁顯示「無法取得版本」| 裝置連不到 GitHub | 檢查網路,可能是 DNS / 網路 / 防火牆問題 |
| 「開始更新」後一直 0% | 下載卡住 | 等 5 分鐘;不行就重啟裝置再試 |
| 更新失敗 | 韌體損毀 / 供電不足 | 重啟裝置(會走舊版),確認供電後重試 |
| 更新後 Apple Home 看到 PID 變化警告 | 重大 PID 改版才會 | 屬正常;按提示重新接受裝置即可,配對不會掉 |

<h2 style="color:#1c3d5a;border-bottom:3px solid #ff6f48;padding-bottom:8px;margin-top:48px;font-size:28px">11. 安全使用</h2>

<div style="background:#fff7f4;border:1px solid #ffd0c5;border-radius:14px;padding:18px;margin:16px 0">

- ⚠️ **AC 110V/220V 高壓側請務必由合格電工安裝**,觸電會致命
- 🔌 接線 / 拆卸**先斷電**,確認電壓符合機型
- 🛠 機殼內無使用者可維修零件,**請勿自行拆解**
- 🌡 工作環境溫度 -10–50 °C
- 🚿 浴室安裝請避開蓮蓬頭直噴範圍;**完全曝雨環境不行**
- 📻 24 GHz mmWave 雷達為**極低功率毫米波**,功率符合 NCC / FCC 民用設備規範,**對人體無害**(類似 Wi-Fi 路由器級別)
- 🔥 繼電器額定 AC 250 V / 10 A,**接的燈具或風扇單體不得超過 10 A 額定電流**
- 📐 機身外殼為阻燃 ABS,**勿覆蓋衣物 / 紙張等可燃物**

</div>

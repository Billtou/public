## 1. 產品簡介

WL-01 是 AUTOMATE 推出的水塔水位自動補水控制器，使用 **超音波 ModBus 水位感測器** + **ESP32-S3 主控** + **雙繼電器**，全程 **Matter over Wi-Fi** 接入 Apple Home / Google Home / Home Assistant / Amazon Alexa，**不需專屬 Hub**。

主要特色：

- **雙網路備援** — 同時支援 **Wi-Fi 與乙太網線**（W5500 SPI），網線優先，網線不通自動切 Wi-Fi
- **Matter 標準協定** — 一次配對在多個生態同時使用（Multi-Admin，最多 5 個 Fabric）
- **本地 Web UI** — 不依賴雲端，區網直連調整滿水/補水閾值、定時補水時間、繼電器同步、OTA 等
- **OLED 即時顯示** — 128×64 SH1106，顯示水塔狀態（自動/停止/加水/異常）+ 水位數據
- **狀態保護** — 感測器斷線自動進「異常」模式停止補水，避免空轉燒馬達
- **OTA 雲端更新** — 廠商發布新版自動偵測，一鍵更新

### 1.1 規格速覽

| 項目 | 內容 |
|---|---|
| 主控 MCU | ESP32-S3-WROOM-1-N4 |
| 通訊協定 | Matter 1.x over Wi-Fi |
| Wi-Fi | 2.4 GHz 802.11 b/g/n（不支援 5 GHz） |
| 乙太網路 | W5500 SPI 10/100 Mbps（RJ45） |
| BLE | 5.0（僅配對用） |
| 水位感測 | ModBus RTU RS485（超音波，0.75-3 m 量程） |
| 繼電器 | ×2，馬達控制 + 備用，Max 10A 250VAC |
| 顯示 | 128×64 SH1106 OLED（單色綠） |
| 物理按鍵 | ×3（加水 / 停止·自動 / 重置） |
| 供電 | DC 12V / 1A |
| 工作環境 | 0–50 °C / 10–90% RH（無凝結） |

### 1.2 包裝內容

- WL-01 主機 × 1
- 超音波水位感測器（ModBus RTU） × 1
- 12V 1A 電源變壓器 × 1
- 安裝快速入門卡（含 Matter QR Code 與 11 碼配對碼） × 1

## 3. OLED 螢幕顯示

### 3.1 版面說明

```
┌────────────────────────────────────┐
│ ████████░░░░░░░░░░░░░░░░░░░░░░░░░░ │  ← 頂部水位進度條（橫向 0-100%）
│ ┌──┐                              │
│ │自│        最高水位 55cm         │  ← 12pt 資訊行 ×3
│ │動│        現在 38cm / 69%       │
│ │  │        加水 60% / 滿 90%     │
│ └──┘                              │
└────────────────────────────────────┘
   ↑ 26pt 大字
   (狀態指示)
```

- **左側 26pt**：兩字大字顯示**目前運作狀態**（自動 / 停止 / 加水 / 異常）
- **右側 12pt 三行**：
  - 行 1：你設定的「水塔最高水位」(cm)
  - 行 2：感測到的「現在水位」(cm) + 對應百分比
  - 行 3：你設定的「補水線 / 滿水線」(%)
- **頂部進度條**：以「最高水位」為 100% 的水塔水量視覺化

### 3.2 4 種運作狀態

OLED 左側 26pt 大字會顯示以下其中一組：

| 螢幕顯示 | 含義 | 馬達 | 自動補水 |
|---|---|---|---|
| **自/動** | 自動模式 — 等待水位低於補水線時自動啟動 | 待機 | ✅ 啟用 |
| **加/水** | 補水中 — 馬達運轉到滿水線自動停止 | 運轉 | ✅ 進行中 |
| **停/止** | 暫停模式 — 維護 / 停水 / 出國時用 | 強制 OFF | ❌ 停用 |
| **異/常** | 感測器異常 — 讀不到水位 | 強制 OFF | ❌ 停用 |

> **異常狀態自動觸發條件**：水位感測器連續多次讀取失敗（線路鬆脫、感測器故障）。請檢查 RS485 接線並重啟裝置。

## 4. LED 狀態指示

依優先級從高到低：

| 優先級 | 狀態 | LED 行為 | 含義 |
|---|---|---|---|
| 1 | 重置警告 | 5 Hz 快閃（亮 0.1s 暗 0.1s） | 重置鍵已按 5 秒，再 5 秒進入工廠重置 |
| 2 | 等待 Wi-Fi | 1 Hz 慢閃（亮 0.5s 暗 0.5s） | 尚未配對 / Wi-Fi 連不上 |
| 3 | 配對中 | 雙閃（快兩下停一下） | Wi-Fi 已連，Matter commissioning 進行中 |
| 4 | 正常運作 | 熄滅 | 一切就緒 |

> **網線優先**：插上網線後，即使 Wi-Fi 沒設定，LED 也會直接從「慢閃」走 commissioning。配對流程仍需 BLE，所以還是要拿手機靠近裝置。

## 5. 按鍵操作

### 5.1 日常按鍵

| 按鍵 | 短按行為 | 觸發狀態 |
|---|---|---|
| **②（綠）加水** | 立即啟動馬達補水 | 切到「加水」狀態 |
| **①（紅）停止 / 自動** | 切換 RUN→STOP→AUTO→STOP→AUTO… | 循環切換 |

**安全保護**：
- 「加水」按下時，若**水位已達滿水線**或**狀態為異常**，馬達不會啟動（避免溢出 / 空轉）
- 「停止/自動」在補水中按下 → 立即停馬達切到「停止」
- 再按一次 → 從「停止」切「自動」（恢復自動補水邏輯）
- 上電後預設一律進入「停止」模式（停電後來電不會自己跑馬達，保護用）

### 5.2 重置鍵

針孔重置鍵，用細針戳入：

| 動作 | 結果 |
|---|---|
| 按住 5–9 秒（LED 從慢閃變快閃時鬆開） | 軟重啟 |
| 按住 ≥ 10 秒（LED 從快閃變熄滅後鬆開） | **工廠重置** — 清除 Matter 配對與 Wi-Fi 帳密，自動重啟 |

> 重置範圍見[章節 13](#13-工廠重置)。

## 6. 接入智能生態（初次配對）

### 6.1 配對前確認

- [ ] 路由器 **Wi-Fi 2.4 GHz** 可用（不支援 5 GHz）
- [ ] 手機 **藍牙開啟**，跟 WL-01 距離 < 1 公尺
- [ ] 手機**已加入** Matter 平台（Apple Home / Google Home / SmartThings / Alexa）
- [ ] 至少有一台 Matter Controller 在家：
  - **Apple**：iPhone / iPad / HomePod / Apple TV（iOS 16.1+）
  - **Google**：Nest Hub / Android 14+ 手機
  - **Home Assistant**：主機 / iPhone / Android 14+ 手機
  - **Alexa**：Echo Hub / Echo (4th gen) / Echo Show

### 6.2 Apple Home

1. 通電 WL-01，等 LED 開始**慢閃**
2. iPhone 開「家庭」App → 右上角 **+** → **加入配件**
3. 對準 QR Code 掃描；或選「沒有代碼/無法掃描」→ 輸入 `3642-630-6820`
4. iPhone 自動傳 Wi-Fi 憑證 → LED 變**雙閃**
5. 配對完成 → LED **熄滅**，出現 **3 個插座 + 1 個濕度感測器**（章節 11 說明用途）

### 6.3 Home Assistant

1. 通電，等 LED 慢閃
2. 開 Home Assistant App → 設定 → Matter → 新增裝置 → **全新的裝置**
3. 掃 QR 或輸入 `3642-630-6820`
4. App 自動完成；HA 自動建立 4 個 entity

### 6.4 Google Home

1. 通電，等 LED 慢閃
2. 開「Google Home」App → 右上 **+** → **設定裝置** → **新裝置** → 選擇家庭
3. App 自動偵測「Matter 裝置」→ 點擊 → 掃 QR 或輸入 `3642-630-6820`
4. 依 App 指示完成 Wi-Fi 設定
5. 配對完成，可在「插座」分類找到

### 6.5 Amazon Alexa

1. 通電，等 LED 慢閃
2. 開 Alexa App → 裝置 → **+** → **新增裝置** → **其他** → **Matter**
3. 掃 QR 或輸入 `3642-630-6820`
4. 完成

## 7. 跨生態接入（Matter Multi-Admin）

### 7.1 機制說明

Matter 規範允許**同一台裝置同時加入多個生態**，最多 **5 個 Fabric**。例如同時被 Apple Home 與 Home Assistant 控制，狀態與動作會在兩個 App 同步。

**關鍵原則**：第二個生態起，**不能用出廠那組 `3642-630-6820`** — 必須由**第一個已配對的生態**產生一組「分享配對碼」（每次新的、有時效）。

### 7.2 從 Apple Home 分享給其他生態

1. 家庭 App → 長按裝置 → **設定**（齒輪圖示）
2. 滑到底，找 **「開啟配對模式」** 或 **「橋接器 > 開啟配對模式」**
3. App 顯示一組 11 碼**拷貝代碼**（15 分鐘內有效）
4. 在第二個生態 App「加入配件」流程輸入這組新碼

### 7.3 從 Home Assistant 分享給其他生態

1. Home Assistant App → 設定 → Matter → 指定轉出裝置 → **分享裝置 → 複製代碼**
2. 在第二個生態 App「加入配件」流程輸入這組新碼

### 7.4 常見誤區

| 誤操作 | 後果 | 正確做法 |
|---|---|---|
| 第二個生態用出廠 `3642-630-6820` | 配對失敗 | 用第一個生態生出來的「分享碼」 |
| 用 Apple 重置裝置後再加入 Google | Apple 那邊也掉了 | 不要重置，用「Multi-Admin 分享」流程 |
| 已加入 5 個生態還想加 | 失敗 | 移除其中一個再加 |
| Wi-Fi 不通就嘗試 multi-admin | 失敗 | 先確認原配對生態正常，Wi-Fi 通才能分享 |

## 8. Web UI 進階設定

### 8.1 連線方式

配對成功後，WL-01 在區網提供 Web UI。打開瀏覽器，網址兩種擇一：

- **直接 IP**：在生態 App 內查看裝置 IP（例 `http://10.0.1.241`）
- **主機名稱**：`http://automate-wl01-XXXXXX.local`（XXXXXX 為機身 MAC 後 6 碼）

> 手機與 WL-01 必須在**同一個 Wi-Fi 網段**。
>
> 網線優先：若有插網線，Web UI 走網線（更穩定）；網線拔掉自動 fallback Wi-Fi。

### 8.2 首頁：即時狀態與控制

| 區塊 | 內容 |
|---|---|
| **裝置資訊** | 韌體版本 / 可用記憶體 / 運行時間 / Matter 配對狀態 / IP / 連線方式（LAN / WIFI） |
| **水位** | 大字顯示 % + 漸變色進度條 + 現在水位 cm |
| **狀態徽章** | 運行狀態（自動/停止/加水/異常）+ 水位感測器（正常/離線） |
| **控制** | 馬達 / 備用繼電器 / 自動化 三個徽章顯示 OFF/ON |
| **動作按鈕** | 「加水」（淡綠）+ 「停止/自動」（淡紅）並排兩顆 |

### 8.3 參數設定

整合卡片，從上而下：

- **滿水線（full）滑桿** — 70-95%，達此水位自動停止補水（預設 90%）
- **補水線（low）滑桿** — 40-80%，低於此水位 -1% 自動啟動補水（預設 60%）
- **水塔最高水位（cm）** — 5-300，**最重要**：百分比 = 現在水位 cm ÷ 最高水位 cm × 100
- **水位偏移校正（cm）** — -20 ~ +20，感測器讀值加偏移
- **⏱ 定時補水** toggle + 時/分下拉 — 開關 OFF 時不執行
- **繼電器 2 同步馬達** toggle — 馬達啟動時備用繼電器同步動作

### 8.4 版更頁：OTA 韌體更新

| 區塊 | 內容 |
|---|---|
| **韌體資訊** | 目前版本、最新版本、檢查狀態 |
| **開始更新按鈕** | 點擊後自動下載 + 安裝 + 重啟（約 1–2 分鐘） |
| **裝置資料清除** | 紅色危險操作，等同硬體長按 10 秒重置 |
| **系統日誌** | 即時 log + RAM / CPU 溫度 |

### 8.5 工程模式

底部 Tab「工程模式」需要登入（預設帳號 `admin` 密碼 `password`，建議首次登入後立即修改）：

- **OEM 白標設定** — 廠商名 / Logo / 主機名前綴
- **使用說明連結覆寫** — 指向你自己的客製文件 URL
- **Matter 配對碼覆寫** — 進階用戶自訂 passcode / discriminator
- **工廠出貨日設定** — 寫入 fctry NVS 的 `mfg_date`

## 9. 必要參數設定

### 9.1 首次安裝必設

依序在 Web UI **參數設定** 卡片完成：

| 參數 | 設定 | 說明 |
|---|---|---|
| **水塔最高水位 (cm)** | **拿尺實際量** | 預設 55cm 通常不對，請量水塔內側從感測器到滿水高度，**這個錯了所有百分比都不準** |
| **水位偏移校正 (cm)** | 通常 0 | 若實測 cm 跟感測器讀的差太多，用偏移補正 |
| **滿水線 (full)** | 預設 90 | 達此 % 自動停止補水。建議 85-90，留緩衝避免溢出 |
| **補水線 (low)** | 預設 60 | 低於此 % 自動啟動補水。建議 40-60，依用水量調 |

> **⚠️ 重要**：若實際最高水位是 200 公分卻誤輸入 300，可能會加到溢出來；敬請準確輸入或比實際**少一些些**的高度，觀察實際運作後再調整。

### 9.2 選用：定時補水

開啟「⏱ 定時補水」toggle 後，從下拉選單選時：分（**分鐘只能選 0/10/20/30/40/50**）：

- 設定例：18：30 → 每天下午 6:30 檢查
- **觸發條件**：到設定時間，且當下水位 < 滿水線 -5%（避免水位夠就跳過）
- **關閉開關 = 不執行定時補水**

### 9.3 選用：水位偏移校正

如果你裝完發現感測器讀 53cm 但實際量是 50cm，把「水位偏移校正」設 -3，讀值就會修正。

### 9.4 選用：繼電器同步

「繼電器 2 同步馬達」開啟後，馬達啟動時，備用繼電器同步啟動。常見用途：
- 同時控制水塔的「補水閥」+「補水指示燈」
- 同時通電加壓泵與電磁閥

## 10. 日常操作

### 10.1 三種模式說明

WL-01 有 3 個常見運作模式 + 1 個保護模式，可從 OLED 螢幕看出：

| 模式 | OLED 顯示 | 行為 |
|---|---|---|
| 自動 | 自/動 | 等水位低於補水線就自動補；達滿水線自動停 |
| 加水中 | 加/水 | 馬達運轉中（可能是自動觸發或手動觸發） |
| 停止 | 停/止 | 完全暫停，不論水位低多少都不補。**出國 / 維護 / 停水時必設** |
| 異常 | 異/常 | 感測器讀不到水位 → 系統強制停馬達避免空轉燒泵 |

### 10.2 自動補水流程

裝好參數後切到「自動」（按 ① 紅鍵直到 OLED 顯示「自/動」），系統自動跑：

```
水位讀值（每秒 1 次）
    ↓
若 < 補水線(low) -1%   → 切到「加水」、馬達 ON
若 ≥ 滿水線(full)     → 切到「自動」、馬達 OFF
否則                   → 維持當前狀態
```

### 10.3 手動加水

兩種方式擇一：

1. **物理按鍵**：按下②（綠）— 馬達立刻啟動，到滿水線自動停
2. **手機 App / Web UI**：打開「加水」開關 — 同上行為

> 在 Apple Home / HA 中關掉「加水」開關後，**約 1 秒內**會自動再開回 ON — 這是設計行為：自動模式下，水位仍低於補水線時系統會持續補。要真停請改關「自動化」開關（章節 11）。

### 10.4 出國 / 停水期間

按 ① 紅鍵切到「停/止」。這時候：
- 馬達強制 OFF
- 自動補水邏輯停用
- 排程也不執行
- 來電後仍維持「停止」（NVS 記憶）

回家後再按 ① 切回「自動」即可。

## 11. Matter 暴露的 4 個實體

WL-01 在 Matter 端暴露 4 個 entity，分別對應不同功能：

| Apple Home 顯示 | Endpoint | 名稱 | ON / OFF 行為 |
|---|---|---|---|
| 「插座」（無數字，第 1 個） | EP1 | **Automation 自動化** | ON → 進入自動模式 / OFF → 進入停止模式（馬達強制 OFF） |
| 「插座 2」 | EP2 | **Manual Fill 手動補水** | ON → 觸發手動補水（含安全檢查）/ OFF → 停馬達回自動 |
| 「插座 3」 | EP3 | **Relay 02 備用繼電器** | 直接控制備用 GPIO 繼電器（無安全邏輯） |
| 「濕度感測器」 | EP4 | **Water Level 水位百分比** | 借用濕度 cluster 顯示 0-100% 水位（無 ON/OFF） |

> 三個「插座」都是 Matter device type 0x010A（OnOff Plug-In Unit），所以 Apple Home / Google Home 預設都顯示為「插座」。建議**長按重新命名**為「自動化」「手動加水」「備用繼電器」，使用上更直觀。
>
> 「水位」用 humidity_sensor cluster 是借用設計（Matter 沒有 distance / level cluster），實際讀的是百分比，用法跟濕度感測器完全一樣，可做場景觸發（如水位 < 40% 推播通知）。

設定面參數（最高水位 / 滿水線 / 補水線 / 排程 / 偏移 / 繼電器同步）**不暴露**到 Matter，請在 Web UI 調整（章節 8.3）。

## 12. OTA 韌體更新

### 12.1 更新流程

1. 連線 Web UI → 底部 **版更** Tab
2. 系統自動向官方檢查最新版本（每 N 小時自動檢查一次）
3. 若有新版，顯示「有新版本可更新」+ 「開始更新」按鈕
4. 點擊「開始更新」→ 確認對話框 → 自動下載
5. 下載完成自動重啟，Web UI 自動 reload 回首頁

整個過程約 1–2 分鐘，期間裝置會暫時離線。

### 12.2 更新會保留 / 清除什麼

| 項目 | 是否保留 |
|---|---|
| Matter 配對（所有生態） | ✅ 保留 |
| Wi-Fi 帳密 | ✅ 保留 |
| 水塔參數（最高水位/滿水線/補水線/排程） | ✅ 保留 |
| 韌體版本 | ❌ 換新版 |

## 13. 工廠重置

### 13.1 硬體重置

- 細針戳針孔，按住重置鍵 **10 秒以上**
- LED 從快閃變熄滅 → 鬆開 → 自動重啟
- 重啟後 LED 慢閃，等待重新配對

### 13.2 Web UI 重置

- 版更頁 → 「裝置資料清除」card → 「清除 NVS（出廠重置）」
- 確認對話框 → 自動重啟

### 13.3 重置會清除什麼

| 項目 | 是否清除 |
|---|---|
| Matter 配對（所有生態） | ✅ 清除 |
| Wi-Fi 帳密 | ✅ 清除 |
| 水塔參數 | ❌ **不清除**（fctry NVS 保留） |
| 韌體版本 | ❌ 不變 |
| 裝置認證資料 / DAC | ❌ 不變（內建出廠憑證，不會掉） |

> 重置後跟全新開封一樣，需重新配對。但**水塔參數會保留** — 不用重設最高水位等設定。
>
> 若要連水塔參數一起清除（例如送修 / 轉手），需在工程模式做「完整重置」。

## 14. 故障排除

### 14.1 配對相關

| 現象 | 原因 | 處理 |
|---|---|---|
| LED 一直慢閃，App 找不到裝置 | 配對流程沒走完 / 手機藍牙關閉 / 距離太遠 | 確認手機藍牙開、距離 < 1 m、Wi-Fi 是 2.4 GHz |
| 雙閃很久後又回慢閃 | 裝置中途重啟（供電不足） | 確認 12V 1A 變壓器 + 換較粗電源線 |
| App 顯示「無法加入」 | 配對碼錯誤 / Fabric 已滿 | 重新輸入碼；若已用 5 個生態，先移除其一 |
| 第二個生態加不進 | 用了出廠那組 `3642-630-6820` | 改用第一個生態的「分享碼」（章節 7） |

### 14.2 運作相關

| 現象 | 原因 | 處理 |
|---|---|---|
| OLED 顯示「異/常」、馬達不動 | ModBus 感測器斷線 | 檢查 RS485 A/B 接線、確認感測器 5V 供電；正常後自動回「自動」狀態 |
| 馬達不停一直加水 | 滿水線設太高 / 感測器位置不對 | 降低滿水線到 85-90%；確認感測器裝在「滿水時水會碰到的高度上方」 |
| 水位百分比明顯不準 | 「水塔最高水位」設錯 | 拿尺實際量水塔內側深度，重設參數 |
| 水位顯示「現在 38cm」但實際不對 | 感測器位置 / 偏移設錯 | 用「水位偏移校正」加減 cm 修正 |
| App 中裝置顯示離線 | Wi-Fi 斷線 / 路由器重啟 | 等 1–2 分鐘自動回連；不行就重新通電；插網線可避免 |
| 關「加水」後 1 秒就自己回 ON | **設計行為**（自動模式下水位仍低） | 要真停請改關「自動化」開關 |

### 14.3 OTA 相關

| 現象 | 原因 | 處理 |
|---|---|---|
| 版更頁顯示「無法取得版本」 | 裝置連不到 GitHub | 檢查網路；可能是 DNS / 網路 / 防火牆問題 |
| 「開始更新」後一直 0% | 下載卡住 | 等 5 分鐘；不行就重啟裝置再試 |
| 更新失敗 | 韌體損毀 / 供電不足 | 重啟裝置（會走舊版），確認供電後重試 |

## 15. 安全使用

- **強電接點警告** — 馬達繼電器接點為 AC 250V / 10A 級。**安裝前務必斷電**，請由有電工資格的人員施作或諮詢專業
- **馬達保護** — 自動模式下「異常」狀態會強制停馬達，但**仍建議在馬達側裝過載保護器**（熱繼電器 / MCB）
- **水塔附近** — 主機請**遠離溢水區**，建議裝在水塔上方乾燥處或控制箱內
- **電源** — 使用原廠 12V 1A 變壓器；換成其他規格供電會影響穩定性
- **戶外使用** — 主機本體**不防水**，戶外請另購防水殼
